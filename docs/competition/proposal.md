# MoonDepSolve 项目申报书

## 1. 项目名称

MoonDepSolve：MoonBit 包生态语义版本与依赖求解工具

## 2. 项目简介

MoonDepSolve 是一个面向 MoonBit 包生态的基础软件组件，提供语义版本解析、版本约束匹配、依赖图求解、冲突诊断和锁文件格式化能力。项目不依赖外部服务，使用纯 MoonBit 实现，可作为包管理器、构建系统、依赖审计工具和自动化发布流程中的底层模块。

当前有效仓库链接如下：

- GitLink 仓库：<https://gitlink.org.cn/python123/moonloglens>
- GitHub 仓库：<https://github.com/python123-ops/moonloglens>

## 3. 项目方向与适用场景

依赖求解是包管理和构建系统中的成熟基础领域。随着 MoonBit 包生态增长，工具链需要能够判断版本兼容性、选择可用版本组合、解释依赖冲突，并形成稳定的解析结果。MoonDepSolve 聚焦这一通用问题，为后续包索引、构建规划、依赖升级、兼容性检查和锁文件生成提供可复用内核。

适用场景包括：

- 包管理器在安装或升级依赖时选择兼容版本。
- 构建工具根据依赖图生成确定性构建计划。
- 项目维护者定位版本冲突来源。
- 自动化发布系统检查依赖范围是否可解析。
- 教学或生态示例展示 MoonBit 中基础软件组件的实现方式。

## 4. 计划实现的核心功能

MoonDepSolve 首版实现以下核心能力：

- 解析 `1.2.3`、`1.2.3-alpha.1` 等语义版本。
- 比较 major、minor、patch 和 prerelease 顺序。
- 解析 exact、caret、tilde、comparator set、wildcard 五类版本约束。
- 用数组结构表示包、版本、依赖和仓库索引，降低依赖风险。
- 对根依赖和传递依赖执行确定性求解，默认选择最高兼容版本。
- 在版本不兼容时返回包含依赖路径的冲突说明。
- 输出稳定的 lock-style 文本，便于测试、展示和后续扩展。
- 提供 CLI 示例、单元测试、README、设计文档和验收清单。

## 5. 原创性说明

本项目为原创 MoonBit 实现，不是移植已有开源项目。项目围绕 MoonBit 包生态中常见的版本约束和依赖图问题重新设计数据结构、解析器、求解流程和错误诊断。首版刻意保持内核小而完整，不引入外部依赖，不绑定具体平台接口，保证可测试、可阅读、可扩展。

## 6. 技术路线

项目采用“解析层 + 约束层 + 求解层 + 展示层”的结构：

- 解析层负责语义版本和版本约束字符串的词法解析。
- 约束层将版本范围统一表示为 comparator 数组，并提供 `matches` 判断。
- 求解层从根依赖开始遍历包索引，选择最高兼容版本，并继续展开传递依赖。
- 诊断层在包缺失、无匹配版本或已选版本冲突时输出可读错误。
- 展示层通过 CLI 和 lock-style 文本展示解析结果。

这种分层方式便于后续扩展到真实包索引读取、锁文件解析、升级建议、依赖图可视化和构建计划生成。

## 7. 预期成果

项目交付后应具备以下成果：

- 一个可运行、可测试的 MoonBit 依赖求解核心库。
- 覆盖版本解析、约束匹配、传递依赖、冲突诊断和输出稳定性的测试集。
- 可通过 `moon run cmd/main` 运行的命令行示例。
- 可在 GitLink 与 GitHub 上验证的完整开源仓库。
- 面向比赛初审的 PDF 项目申报书。

## 8. 后续扩展方向

MoonDepSolve 后续可继续扩展：

- 支持从包索引文件读取真实依赖数据。
- 生成结构化锁文件并支持读回。
- 给出可升级版本建议和最小变更升级计划。
- 输出依赖图和冲突图。
- 与 MoonBit 构建流程或包发布流程集成。

## 2. 项目简介

MoonDepSolve 面向 MoonBit 包生态中的版本约束和依赖选择问题，提供语义版本解析、版本范围匹配、依赖图求解、冲突诊断和依赖锁定结果输出。项目使用 MoonBit 编写，不依赖外部服务，适合在包管理、构建规划、依赖检查和自动化发布流程中作为基础组件使用。

当前有效仓库链接如下：

- GitLink 仓库：<https://gitlink.org.cn/python123/moondepsolve>
- GitHub 仓库：<https://github.com/python123-ops/moondepsolve>

## 3. 项目方向与适用场景

依赖求解是包管理和构建系统中的基础能力。随着 MoonBit 包数量增加，工具链需要能够判断版本兼容性、选择可用版本组合、解释依赖冲突，并形成稳定的解析结果。MoonDepSolve 聚焦这一问题，为包索引、构建规划、依赖升级、兼容性检查和锁文件生成提供一个可复用的 MoonBit 实现。

适用场景包括：

- 包管理器在安装或升级依赖时选择兼容版本。
- 构建工具根据依赖图生成确定性构建计划。
- 项目维护者定位版本冲突来源。
- 自动化发布系统检查依赖范围是否可解析。
- 教学或生态示例展示 MoonBit 中基础软件组件的实现方式。

## 4. 计划实现的核心功能

MoonDepSolve 目前实现以下核心能力：

- 解析 `1.2.3`、`1.2.3-alpha.1` 等语义版本。
- 比较 major、minor、patch 和 prerelease 顺序。
- 解析 exact、caret、tilde、comparator set、wildcard 五类版本约束。
- 用数组结构表示包、版本、依赖和仓库索引，降低依赖风险。
- 对根依赖和传递依赖执行确定性求解，默认选择最高兼容版本。
- 在版本不兼容时返回包含依赖路径的冲突说明。
- 输出稳定的依赖锁定结果，便于测试、展示和后续处理。
- 提供 CLI 示例、单元测试、README、设计文档和验收清单。

## 5. 实现说明

项目代码围绕 MoonBit 包生态中常见的版本约束和依赖图问题自行设计，包括数据结构、解析器、求解流程和错误诊断。目前版本控制在较小规模内，不引入外部依赖，也不绑定具体平台接口，便于评审直接克隆、运行和检查。

## 6. 技术路线

项目采用“解析层 + 约束层 + 求解层 + 展示层”的结构：

- 解析层负责语义版本和版本约束字符串的词法解析。
- 约束层将版本范围统一表示为 comparator 数组，并提供 `matches` 判断。
- 求解层从根依赖开始遍历包索引，选择最高兼容版本，并继续展开传递依赖。
- 诊断层在包缺失、无匹配版本或已选版本冲突时输出可读错误。
- 展示层通过 CLI 和依赖锁定结果文本展示解析结果。

这种分层方式便于继续接入真实包索引、锁文件解析、升级建议、依赖图输出和构建计划生成。
